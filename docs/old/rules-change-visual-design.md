# Visual Design Change Rules

この文書を、このリポジトリで画面設計と視覚表現を変更する時に参照する正本として扱わなければならない。ここへ記述してよい内容は、可読性、余白、文字組み、色の使い方などの視覚設計の共通基準だけに限定し、それ以外の仕様や詳細設計を混在させてはならない。

文書群の役割分担と文書運用ルールを判断する場合に限って [rules-update-documents.md](./rules-update-documents.md) を参照しなければならない。視覚設計そのものの判断に該当しない限り、他の rules を代用参照してはならない。

## 優先順位

- 視覚設計を判断する時は、情報密度より可読性を優先しなければならない。
- 視覚設計を判断する時は、局所最適より一貫性を優先しなければならない。

## 既定値

- 余白を定める時は `8pt` グリッドを基本単位にしなければならず、`4pt` は補助調整に限定しなければならない。これ以外の単位を常用してはならない。
- 文字サイズを定める時は `body` を基準に Dynamic Type で拡縮しなければならず、本文可読性を先に守らなければならない。

## 避けること

- 情報量の多い画面では、card の中にさらに card を重ねて視覚的な入れ子を深くしてはならない。
- primary color を大面積の背景色として多用してはならない。

## レイアウト補足

- レスポンシブな差分を導入する場合は、機能差分ではなく表現差分へ閉じ込めなければならない。機能差分として拡散させてはならない。
- 分割表示や横幅拡張を行う場合は、情報量を増やしすぎてはならず、読み順と操作の見通しを優先しなければならない。
- 視覚表現を調整する場合は、装飾よりも視認性、操作対象の識別、長時間閲覧時の疲れにくさを優先しなければならない。
