# Design Rules

この文書は、このリポジトリの画面設計と視覚表現に関する上位ルールを定める正本である。ここでは、可読性、余白、文字組み、色の使い方など、視覚設計の共通基準を扱う。

文書群の役割分担と文書運用ルールは [rules-document.md](./rules-document.md) を参照する。

## 優先順位

- 情報密度より可読性を優先する。
- 局所最適より一貫性を優先する。

## 既定値

- 余白は `8pt` グリッドを基本単位とし、`4pt` は補助調整に限定する。
- 文字サイズは `body` を基準に Dynamic Type で拡縮し、本文可読性を先に守る。

## 避けること

- 情報量の多い画面で card の中にさらに card を重ね、視覚的な入れ子を深くしない。
- primary color を大面積の背景色として多用しない。

## レイアウト補足

- レスポンシブな差分は機能差分ではなく表現差分へ閉じ込める。
- 分割表示や横幅拡張では情報量を増やしすぎず、読み順と操作の見通しを優先する。
- 装飾よりも、視認性、操作対象の識別、長時間閲覧時の疲れにくさを優先する。
