# Session Start Rules

この文書は、開発セッション開始という 1 つのタスクだけを定義する完結文書である。

## タスク定義

- セッション開始とは、作業開始前の基準状態を整えるタスクである。
- セッション開始タスクは、次の 2 条件を満たした時だけ完了とみなす。

## 完了条件

- `main` が最新であり、その最新を基準に本日の日付のブランチへ checkout されていること。
  - 当該のブランチが存在する場合は、そのブランチの最新へ checkout されていること。
  - 当該のブランチが存在しない場合は、最新の `main` を起点に本日の日付のブランチを作成し、そのブランチへ checkout されていること。
- すべての `*-latest.md` 文書について、次を保証すること。
  - 本日以外の日付見出しを含む内容は、`scripts/rotate-history` を通して対応する `*-log.md` へ掃き出されていること。
  - `*-latest.md` の 1 行目が本日の日付見出しであること。

## 禁止事項

- 上記 2 条件以外の作業を、この文書の対象へ含めてはならない。
- セッション開始タスクが未完了のまま、以後の開発タスクへ進んではならない。
- `*-log.md` を直接読み込んではならない。
- 履歴移動を手作業や別経路で行ってはならず、`scripts/rotate-history` を使わなければならない。
- 履歴移動後の差分確認が必要な場合は、`git` コマンドを通して確認しなければならない。
